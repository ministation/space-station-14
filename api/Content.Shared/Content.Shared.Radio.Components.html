Classes

ActiveRadioJammerComponent

Prevents all radio in range from sending messages

EncryptionKeyComponent

This component is currently used for providing access to channels for "HeadsetComponent"s. It should be used for intercoms and other radios in future.

EncryptionKeyHolderComponent

This component is by entities that can contain encryption keys

HeadsetComponent

This component relays radio messages to the parent entity's chat when equipped.

IntercomComponent

Handles intercom ui and is authoritative on the channels an intercom can access.

RadioJammerComponent

When activated (ActiveRadioJammerComponent) prevents from sending messages in range Suit sensors will also stop working.

TelecomExemptComponent

This is used for a radio that doesn't need a telecom server in order to broadcast.

TelecomServerComponent

Entities with TelecomServerComponent are needed to transmit messages using headsets. They also need to be powered by have EncryptionKeyHolderComponent and filled with encryption keys of channels in order for them to work on the same map as server.
